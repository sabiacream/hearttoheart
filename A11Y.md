# Heart to Heart 111 — accessibility audit (April 2026)

This pass focused on **WCAG 2.1 AA** contrast for normal text (4.5:1) and large/bold text (3:1), **meaningful image alternatives**, **keyboard flow** through the reading, and **names for controls**. Visual redesign was avoided; adjustments use design tokens and targeted CSS.

---

## 1. Contrast — findings & fixes

### Automated checks (approximate)

Evaluations used **sRGB relative luminance** vs representative backgrounds:

| Area | Issue | Approx. ratio before | Fix |
|------|--------|----------------------|-----|
| Hero trust strip, separators | `--ts` / `--tmu` on parchment `#f4eadc` | ~3.4 / ~2.8 | Critical CSS + tokens: `--text-soft-parchment`, `--text-muted-parchment` |
| Hero tag (uppercase gold) | `#b89452` on parchment | ~2.4 | `var(--gold-text-parchment)` (`#7a5c28`, ~5.2:1) |
| Hero `.brand .to` | `#8f7a71` on parchment | ~3.4 | `var(--text-soft-parchment)` |
| Hero location line | `#8b756d` on parchment | ~3.6 | `var(--text-muted-parchment)` |
| Ritual UI (`--ts` / `--tmu` on cream panels) | Same tokens used for footer + panels | Mixed | **Split:** `--ts-on-light` / `--tmu-on-light` for parchment; `--text-soft-footer` / `--text-muted-footer` for legacy light footer rules |
| Site footer disclaimer (`.fdi`, `.footer-legal-hint`) | Very light translucent text on `#0a0812` | **~2.6–3.3** | Increased alpha to **0.5** on primary disclaimer copy; legal links **0.62** |
| Site footer `.fco` | `rgba(247,242,234,.45)` | ~4.1 (borderline) | **0.5** |
| Studio review stars (`.rev-s`) | Translucent gold on parchment | ~1.8 (decorative + `sr-only` duplicate) | Solid `var(--gold-text-parchment)` for visible stars |
| Studio review attribution (`cite.rev-n`) | `rgba(90,79,86,.72)` on parchment | **~3.5** | Solid `var(--text-muted-parchment)` |

### Verified OK (no change)

- **Password gate** (`h2h-deferred.css`): panel copy, labels, and hints on dark / glass stack meet **≥4.5:1** when composited against `#07050d`.
- **Charm labels** (`.trk .tn`), **spread descriptions** (`.sp .sd`), **category chips** (`.qcat` uses `--tm` on white): meet AA after `--tmu-on-light` / existing `--tm`.
- **Inner pages** (`about`, `faq`, `contact`, `restore`, `terms`): `main.parlor-panel` body uses `--parlor-ink` / `--parlor-muted` on parchment; sampled ratios **≥5:1** for body muted text.
- **Body text on midnight** (`inner-parlor`): `rgba(247,242,234,0.84)` over `#0a0812` composites to **~12.5:1**.

### Token changes (`assets/css/design-system.css`)

New / updated:

- `--text-soft-parchment: #6b5c55`
- `--gold-text-parchment: #7a5c28`

- `--ts-on-light` / `--tmu-on-light` — aliases to parchment tokens for ritual surfaces

## 3. Keyboard navigation (password gate → unlock)

| Topic | Result |
|--------|--------|
| **Password modal** | `role="dialog"` `aria-modal="true"`; **Tab** cycles panel focusables; **Shift+Tab** wraps; `activatePwGateModal` sets `inert` on `.wrap` / `#hbg` so focus stays in the gate. |
| **Escape** | Announces closure message; does not dismiss (by design — phrase required). |
| **After unlock** | Focus moves to `#main-content`. |
| **Charm arena** | `inert` until chest opens; removed in JS when chest opens so trinkets are tabbable. |
| **Spread** | `role="radiogroup"`; `.sp` buttons keyboard operable. |
| **Cards** | `.tc` buttons flip on Enter/Space (native button behavior). |
| **Preview / PayPal** | `.ppb` buttons and links in tab order; focus rings from `assets/css/polish.css` + deferred. |
| **Traps** | None observed in markup/JS review; `inert` cleared when gate closes. |

## 6. Residual notes

- **Large text**: Some uppercase labels (e.g. `.65rem` eyebrow) may qualify as non-large under WCAG; tokens were chosen to meet **4.5:1** where used for supporting copy.
- **Third-party**: PayPal checkout UI inside its iframe is governed by PayPal’s own accessibility.
- **Retest**: Re-run contrast after any background image or opacity changes to hero/footer.
