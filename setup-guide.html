Private Launch Setup

This is the private checklist for getting the live paid tarot website online safely. Keep this document off the public website.

What Is Already Done

  • The website presents a paid tarot reading flow, not a free unlocked tool.
  • The reading session saves before checkout starts.
  • Each session uses one payment for one reveal.
  • Customers can keep a reading code if anything goes wrong.
  • Owner-only setup notes were removed from the public page.
  • The public demo unlock path is disabled.

Recommended Payment Stack

Use PayPal for the cleanest live setup on this site.

  • Create the PayPal checkout on the server.
  • Use a PayPal webhook to confirm payment.
  • Redirect the customer back to the website after payment.
  • Only unlock the full reading after the server says the session is paid.

Required Setup

  1. Open index.html and find the CONFIG block.
  2. Leave debugAutoUnlock set to false for the live website.
  3. Add createCheckoutEndpoint for the PayPal checkout flow.
  4. Add statusEndpoint so the site can verify whether the session is paid before reveal.
  5. Do not launch without both endpoints connected. This avoids paid-but-locked support issues.
  6. Set your PayPal success URL to return the customer to the website with ?reading={sessionId}.
const CONFIG = {
  priceDollars: 5,
  createCheckoutEndpoint: 'https://yourdomain.com/api/create-checkout',
  statusEndpoint: 'https://yourdomain.com/api/payment-status',
  debugAutoUnlock: false
};

Success URL Pattern

https://yourdomain.com/?reading={sessionId}

The page uses that reading code to look up the paid status after checkout.

Endpoint Contract

createCheckoutEndpoint

Receives the reading session info and returns a PayPal checkout URL.

Request body:
{
  "sessionId": "H2H-20260307-ABCD",
  "amountDollars": 5,
  "draft": {
    "keepsake": "heart",
    "moon": "new",
    "zodiac": "pisces",
    "spread": 3,
    "question": "What do I need to know right now?"
  },
  "preview": {
    "title": "Past, present, future"
  }
}

Response:
{
  "checkoutUrl": "https://www.paypal.com/checkoutnow?token=..."
}

statusEndpoint

Returns the current payment state for a session.

Response:
{
  "status": "paid",
  "paidCredits": 1
}

Only the server should mark a session as paid. Do not trust the front end by itself.

PayPal Version

  1. Create a PayPal checkout flow for one tarot reading.
  2. Your server creates the PayPal order when the website calls createCheckoutEndpoint.
  3. Set the success URL to return to your website with the session ID in the query string.
  4. Use a PayPal webhook to mark the reading session as paid in your database.
  5. Have statusEndpoint read that database record and return the paid state.

Before Launch Checklist

  • Start a reading and confirm a reading code appears immediately.
  • Turn the cards first and confirm the teaser appears before PayPal opens.
  • Finish a live or test PayPal payment and confirm the full reading unlocks only after payment verification.
  • Confirm the same session cannot be used to unlock the full reading more than once.
  • Confirm the restore box works in the same browser.
  • Confirm the Facebook link goes to the correct page.
  • Confirm the support phone number is correct.
  • Review the disclaimer text one final time.

If You Want The Safest Launch

The safest version is:

  1. Server creates checkout.
  2. Customer sees the teaser first, then PayPal opens.
  3. PayPal confirms payment by webhook.
  4. Server marks that session as paid.
  5. Website checks paid status.
  6. Website opens the full reading automatically.

That avoids the biggest support problem: someone paying successfully but the website failing to unlock cleanly.
